# Repository Guidelines

## プロジェクト構成と配置
- 静的サイト。各ページはリポジトリ直下にあり、メインは `top.html`（アンカーは `#voice` `#guide` `#access`）。補助ページは `about.html` `activities.html` `support.html` `voices.html` と各ライフステージガイド (`guide-infant.html` `guide-school.html` `guide-youth.html` `guide-adult.html`)。
- `style.css` に共通スタイルがまとまっています。下書き用の `HP改修案_v2.html` と `test.html` は削除済みで公開導線に含めないでください。
- 画像などの新規アセットを置く場合は HTML と同階層に保存し、相対パスで参照します（現状はテキストと絵文字のみ）。ファイル名はハイフン区切りの半角英数字で統一してください。
- ページ間のリンクはトップからたどれるよう維持します。孤立ページを作った場合は必ずナビゲーションやフッターに導線を追加します。

## デザイン方針・トンマナ
- コンセプト: 「孤立させない家族支援 - 経験と共感でつなぐ」。ターゲットは障害児者を育てる母親中心（父親も主体）、寄付・入会・相談の導線強化。
- トーン: 親しみやすい・温かい・前向き。安心感と信頼を優先し、断定的/専門用語過多は避ける。
- カラー: メイン淡グリーン `#81C784`、濃グリーン `#519657`（文字/強調）、アクセントオレンジ `#FFB74D`、サブブルー `#4FC3F7`、ベース `#FFFCF5`、テキスト濃グレー `#2C2C2C`。コントラスト確保。
- フォント: Google Fonts の `BIZ UDPGothic` と `Noto Sans JP` を基本。文字サイズは読みやすさ重視（18px目安）。
- ビジュアル: 笑顔の家族・活動シーン、丸みのあるイラストや絵文字で温度感を出す。顔が見える安心感を意識。
- CTA配置: PCヘッダーとスマホ固定CTAに「寄付・入会」「相談・問い合わせ」。全ページで迷わせない。
- アクセシビリティ: JIS X 8341-3 を意識し、文字サイズ・コントラスト・フォーカス可視性を担保。2024年4月義務化の合理的配慮に留意。

## 情報アーキテクチャの骨子
- トップ: ヒーロー（ミッション）→知りたい情報（体験談/制度/交流）→年齢別ガイド→私たちについて→フッター（サイトマップ/連絡先/寄付導線）。
- 体験談・ピアサポートを最上位の安心コンテンツとして扱う。次点で制度・お金、活動内容。
- 年齢別ガイド: 乳幼児/学齢/青年/成人の4段階。各ページに前後ナビとトップへの戻り導線。
- CTAは常時到達できるようナビ・固定フッター・フッターに重複配置。

## ビルド・テスト・開発コマンド
- ビルド不要。ブラウザで `top.html` を直接開いて確認できます。
- 相対リンク確認のためローカルサーバー推奨: `python3 -m http.server 8000` をリポジトリ直下で実行し、`http://localhost:8000/top.html` を開く。
- 変更時はブラウザのハードリロードを行い、キャッシュを避けてください。リンクやCTAを触ったら必ず遷移先とアンカー解決をクリックチェックしてください。
- デザイン確認は最新の Chrome/Safari を優先し、可能ならiOS/Androidの実機またはデバイスエミュレータで幅320px〜1440pxをざっと確認します。
- 紙配布を想定したプリント時の崩れも稀に確認すると、想定外の改行やはみ出しを事前に防げます。

## コーディングスタイルと命名
- HTML: 4スペースインデント、小文字タグ/属性、`lang="ja"` を付与。クラス名はハイフン区切り（例: `.hero-title` `.info-card`）。`.container` `.section-title` `.btn*` など既存のレイアウト/ボタンユーティリティを再利用し、意味的なセクションタグを優先。
- CSS: `style.css` のコメントブロック内に追加。`:root` のカラーパレットを踏襲し、文字は定義済みの Google Fonts を継承。レイアウトは flex/grid を優先し、レスポンシブは既存の `@media (min-width: 769px)` と `@media (max-width: 768px)` に寄せて追記。
- コンテンツ: 家族に寄り添う簡潔な表現を維持し、専門用語や断定的表現を避けます。問い合わせ導線や寄付導線は迷わない位置に配置し、見出しは読者が次に何を知れるか即座に分かる文言にします。
- コードコメントは最小限。再利用するクラスや色を追加した場合は `:root` とセクションコメント付近に簡潔にメモを残すと後続が追従しやすくなります。
- ファイル末尾には余計な空行やBOMを残さないでください。文字コードはUTF-8が前提です。

## 運用・更新指針（コンテンツ拡充）
- トップに「新着/お知らせ」枠を設け、最新3件＋「一覧」導線を想定。新規記事を作ったら必ずトップにも告知。
- 更新リズム目安: イベント予定/レポート月1本、体験談を四半期で2本（診断直後/学校/就労/親なき後をローテ）、制度改定は年度替わりや法改正時に`support.html`更新。
- 記事テンプレ（見出しだけ決めて効率化）: イベント予定（日時/場所/対象/申し込み）、レポ（参加者・写真1枚・気づき1行・次回案内）、制度アップデート（変わった点3 bullet・対象・申請時期・相談先）、体験談（ステージバッジ＋悩み→変化→今の一言）。
- 運用フロー: トピック決定→下書き確認（誤字/リンク）→公開反映（対象ページ＋トップ新着）→http.serverでクリックチェック→更新ログに日時/担当/内容/影響ページを残す。
- 年齢別ガイドの見直しタイミング: 学齢期は毎年2〜3月、成人期は制度改定時、共通でアンカー導線と固定CTAの動作確認をセットで行う。

## テスト指針
- PC/スマホ幅で手動QA。スティッキーヘッダー、モバイル固定CTA、アンカー（`#voice` `#guide` `#access` `#contact` `#donate`）の挙動を確認。
- CSS変更後はタイポグラフィや余白、`--color-base` 上でのコントラストを目視。ボタンのホバー/フォーカスが残っているか、フッターが固定CTAと重ならないかをチェック。可能なら画面読み上げでリンクテキストが意味を持つかも確認。
- 新規ページ追加時は `top.html` への戻りナビや共有ヘッダー/フッターのスタイル崩れ有無を確認。ファイルを増やした場合はリンク切れがないか `http.server` 上で全ページを一巡してください。
- 変更が小さくても、修正箇所をスクリーンショット1〜2枚で残しておくと差分確認と共有が楽になります。
- 配色を変える場合はコントラスト比を意識し、ボタンやリンクは `color-main` か `color-accent` 系の範囲で調整してください。

## コミットとプルリク
- Git を使う場合は命令形の Conventional Commit 推奨（例: `feat: add youth guide cta` `fix: adjust fixed cta spacing`）。
- PR では概要、影響ページ、見た目変更のビフォー/アフター（スクリーンショット推奨）、手動テスト結果（ブラウザ・幅）を簡潔に記載。関連タスクやIssue IDがあればリンクしてください。アクセシビリティ改善や文言変更は意図を明記するとレビューが速くなります。
- 文言修正だけのPRでもレビュワーが迷わないよう、変更理由（誰の不安を解消するか、どの導線を強化するか）を1行で添えてください。

## セキュリティと設定
- 外部ライブラリは未使用。新規で追加する場合はライセンスとCDN可否を確認し、`<head>` に `preconnect` など既存の書き方を合わせてください。
- フォーム追加時は `mailto:` ではなく専用の問い合わせページへのリンクを優先し、個人情報をHTML内に埋め込まないでください。
- 公開前に余計なデバッグ用テキストや一時リンクが残っていないか、HTML全体をざっとスクロールして確認してください。

## 作業のすすめ方
- デザイン方針・トンマナは本ファイル内（「デザイン方針・トンマナ」セクション）に集約。着手前に確認して認識を揃える。
- 変更を始める前に対象HTMLをバックアップコピーせず、こまめに保存・差分確認を行ってください。複数ページへ影響するクラスを変更する場合は影響範囲をメモに残し、着手順を決めてから編集すると安全です。
- 作業完了後は `style.css` の該当箇所に軽いコメントを残し、次に触る人が意図を把握できるようにしてください。
