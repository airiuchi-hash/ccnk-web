# Repository Guidelines

## プロジェクト概要

### 組織情報
- **組織名**: 障害児者の将来を守る父の樹会
- **設立**: 1978年（千葉大学教育学部付属特別支援学校の父兄が中心）
- **理念**: 「子供の生涯の幸せシステム」作り
- **特徴**: 父親も率先して共同で取り組む点が特色

### プロジェクト構成と配置
- 静的サイト。各ページはリポジトリ直下にあり、メインは `top.html`（アンカーは `#voice` `#guide` `#access`）。補助ページは `about.html` `activities.html` `support.html` `voices.html` と各ライフステージガイド (`guide-infant.html` `guide-school.html` `guide-youth.html` `guide-adult.html`)、入会案内 `join.html`、相談窓口 `contact.html`。
- `style.css` に共通スタイルがまとまっています。下書き用の `HP改修案_v2.html` と `test.html` は削除済みで公開導線に含めないでください。
- 画像などの新規アセットを置く場合は HTML と同階層に保存し、相対パスで参照します（現状はテキストと絵文字のみ）。ファイル名はハイフン区切りの半角英数字で統一してください。
- ページ間のリンクはトップからたどれるよう維持します。孤立ページを作った場合は必ずナビゲーションやフッターに導線を追加します。

## デザイン方針・トンマナ

### コンセプトとターゲット
- **コンセプト**: 「孤立させない家族支援 - 経験と共感でつなぐ」
- **ターゲット**: 障害児者を育てる母親中心（父親も主体）、寄付・入会・相談の導線強化
- **ターゲット心理**: 孤独感、将来への不安、多忙感、育児ストレス
- **喚起する感情**: 安心感、共感、信頼性、温かさ、前向きさ
- **デザイン目的**: 会員獲得と寄付金集めを促進する外部向けの情報発信プラットフォームへの転換

### デザインの方向性
- **全体イメージ**: 親しみやすく、柔らかい、温かみのあるデザイン。脱・堅苦しさ/医療的な無機質さ
- **トーン**: 親しみやすい・温かい・前向き。安心感と信頼を優先し、断定的/専門用語過多は避ける

### カラースキーム（実装値）
- **メイン**: 淡いグリーン `#81C784`（安心感）
- **メイン（濃）**: 濃いグリーン `#519657`（文字・強調）
- **アクセント**: 控えめなオレンジ `#FFB74D`（希望）
- **サブ**: 暖色系ブルー `#4FC3F7`（信頼・相談）
- **ベース**: クリーム/ベージュ `#FFFCF5`（温かみ）
- **テキスト**: 濃いグレー `#2C2C2C`（読みやすさ）
- コントラスト確保を優先

### タイポグラフィとビジュアル
- **フォント**: Google Fonts の `BIZ UDPGothic`（ユニバーサルデザインフォント）と `Noto Sans JP` を基本。文字サイズは読みやすさを重視して大きめ（18px基準）に設定
- **ビジュアル**: 笑顔の子どもや家族、活動風景（ピアサポート、学習会）。スタッフの顔が見える安心感を意識
- **イラスト**: 手書き風または丸みを帯びた柔らかいタッチ。「仲間ができる」イメージ

### CTA（行動喚起）配置
- **PC**: ヘッダー内に「寄付・入会」「相談・問い合わせ」ボタンを配置
- **スマホ**: 画面下部に固定フッターとして常時表示
- どのページからでもアクセス可能にする。全ページで迷わせない

### アクセシビリティとユーザビリティ
- **アクセシビリティ（必須）**: 
  - JIS X 8341-3（ウェブアクセシビリティ基盤委員会）を考慮
  - 文字サイズ、コントラスト比の確保（高齢者・弱視者配慮）
  - 合理的配慮の提供（2024年4月義務化対応）
  - フォーカス可視性を担保
- **ユーザビリティ**:
  - シンプルで美しいデザイン
  - 「見て分かる」直感的な操作性
  - 迷わせない明確な階層構造

## 情報アーキテクチャ

> **詳細なページ構成ツリー**: 各ページの詳細な構成ツリー表記は [`sitemap.md`](./sitemap.md) を参照してください。

### ページ構成
- **トップ（top.html）**: ヘッダー（共通ナビ/CTA）→ヒーロー（ミッション）→優先情報3カード（体験談/制度/交流）→体験談ダイジェスト（#voice）→年齢別ガイド一覧（#guide：乳幼児/学齢/青年/成人）→「私たちについて」→フッター→固定CTA（join.html / contact.html）

### グローバルナビゲーション
- **メインナビ**: ホーム / 父の樹会について / 活動内容 / お金と制度 / 体験談 / 年齢別ガイド
- **CTA**: 右に「寄付・入会（join.html）」/「相談・問い合わせ（contact.html）」を固定

### 主要ページ構成
- **入会案内（join.html）**: ヘッダー（共通）→ヒーロー（入会のご案内）→入会の流れ→会費・サポート内容→FAQ→CTA（相談・活動導線）
- **相談・問い合わせ（contact.html）**: ヘッダー（共通）→ヒーロー（まずは相談）→相談方法（電話/フォーム準備中/イベント見学）→よくあるご相談→CTA（入会・活動導線）

### 下層ページ共通仕様
- `about.html` / `activities.html` / `support.html` / `voices.html` / `guide-*.html` は共通ヘッダー/フッター/固定CTA
- **年齢別ガイド**: 乳幼児/学齢/青年/成人の4段階。各ガイドに前後ナビとトップのガイド一覧への戻り導線を置く
- **フッター**: 団体情報 / 知りたい情報 / 年齢別ガイド / ご支援・参加（入会・寄付・相談）を共通化。固定CTAも同じリンク構成

### コンテンツ優先順位
1. **体験談・ピアサポート**: 「一人じゃない」と感じられる共感コンテンツ
2. **制度・費用**: 親御さんが切実に知りたい「生の情報」
3. **活動内容**: 楽しさやメリットが伝わる内容

## 開発・テスト環境

### ビルドとローカルサーバー
- ビルド不要。ブラウザで `top.html` を直接開いて確認できます
- 相対リンク確認のためローカルサーバー推奨: `python3 -m http.server 8000` をリポジトリ直下で実行し、`http://localhost:8000/top.html` を開く（F5で起動可能: `.vscode/launch.json` 参照）
- 変更時はブラウザのハードリロードを行い、キャッシュを避けてください。リンクやCTAを触ったら必ず遷移先とアンカー解決をクリックチェックしてください

### デザイン確認
- 最新の Chrome/Safari を優先し、可能ならiOS/Androidの実機またはデバイスエミュレータで幅320px〜1440pxをざっと確認します
- 紙配布を想定したプリント時の崩れも稀に確認すると、想定外の改行やはみ出しを事前に防げます

### テスト指針
- **基本QA**: PC/スマホ幅で手動QA。スティッキーヘッダー、モバイル固定CTA、アンカー（`#voice` `#guide` `#access`）の挙動を確認。入会/相談は `join.html` / `contact.html` への導線をクリック確認
- **CSS変更後**: タイポグラフィや余白、`--color-base` 上でのコントラストを目視。ボタンのホバー/フォーカスが残っているか、フッターが固定CTAと重ならないかをチェック。可能なら画面読み上げでリンクテキストが意味を持つかも確認
- **新規ページ追加時**: `top.html` への戻りナビや共有ヘッダー/フッターのスタイル崩れ有無を確認。ファイルを増やした場合はリンク切れがないか `http.server` 上で全ページを一巡してください
- **変更記録**: 変更が小さくても、修正箇所をスクリーンショット1〜2枚で残しておくと差分確認と共有が楽になります
- **配色変更**: コントラスト比を意識し、ボタンやリンクは `color-main` か `color-accent` 系の範囲で調整してください

## コーディングスタイルと命名

### HTML
- 4スペースインデント、小文字タグ/属性、`lang="ja"` を付与
- クラス名はハイフン区切り（例: `.hero-title` `.info-card`）
- `.container` `.section-title` `.btn*` など既存のレイアウト/ボタンユーティリティを再利用し、意味的なセクションタグを優先

### CSS
- `style.css` のコメントブロック内に追加
- `:root` のカラーパレットを踏襲し、文字は定義済みの Google Fonts を継承
- レイアウトは flex/grid を優先し、レスポンシブは既存の `@media (min-width: 769px)` と `@media (max-width: 768px)` に寄せて追記

### コンテンツ
- 家族に寄り添う簡潔な表現を維持し、専門用語や断定的表現を避けます
- 問い合わせ導線や寄付導線は迷わない位置に配置し、見出しは読者が次に何を知れるか即座に分かる文言にします

### コード管理
- コードコメントは最小限。再利用するクラスや色を追加した場合は `:root` とセクションコメント付近に簡潔にメモを残すと後続が追従しやすくなります
- ファイル末尾には余計な空行やBOMを残さないでください。文字コードはUTF-8が前提です

## 運用・更新指針

### 新着情報管理
- トップに「新着/お知らせ」枠を設け、最新3件＋「一覧」導線を想定。新規記事を作ったら必ずトップにも告知

### 更新リズム目安
- イベント予定/レポート: 月1本
- 体験談: 四半期で2本（診断直後/学校/就労/親なき後をローテ）
- 制度改定: 年度替わりや法改正時に`support.html`更新

### 記事テンプレート（見出しだけ決めて効率化）
- **イベント予定**: 日時/場所/対象/申し込み
- **レポ**: 参加者・写真1枚・気づき1行・次回案内
- **制度アップデート**: 変わった点3 bullet・対象・申請時期・相談先
- **体験談**: ステージバッジ＋悩み→変化→今の一言

### 運用フロー
- トピック決定→下書き確認（誤字/リンク）→公開反映（対象ページ＋トップ新着）→http.serverでクリックチェック→更新ログに日時/担当/内容/影響ページを残す

### 年齢別ガイドの見直しタイミング
- 学齢期: 毎年2〜3月
- 成人期: 制度改定時
- 共通: アンカー導線と固定CTAの動作確認をセットで行う

## Git運用

### コミット
- Git を使う場合は命令形の Conventional Commit 推奨（例: `feat: add youth guide cta` `fix: adjust fixed cta spacing`）

### プルリクエスト
- PR では概要、影響ページ、見た目変更のビフォー/アフター（スクリーンショット推奨）、手動テスト結果（ブラウザ・幅）を簡潔に記載。関連タスクやIssue IDがあればリンクしてください
- アクセシビリティ改善や文言変更は意図を明記するとレビューが速くなります
- 文言修正だけのPRでもレビュワーが迷わないよう、変更理由（誰の不安を解消するか、どの導線を強化するか）を1行で添えてください

## セキュリティと設定

### 外部ライブラリ
- 外部ライブラリは未使用。新規で追加する場合はライセンスとCDN可否を確認し、`<head>` に `preconnect` など既存の書き方を合わせてください

### フォームと個人情報
- フォーム追加時は `mailto:` ではなく専用の問い合わせページへのリンクを優先し、個人情報をHTML内に埋め込まないでください

### 公開前確認
- 公開前に余計なデバッグ用テキストや一時リンクが残っていないか、HTML全体をざっとスクロールして確認してください

## 作業のすすめ方

### 着手前
- デザイン方針・トンマナは本ファイル内（「デザイン方針・トンマナ」セクション）に集約。着手前に確認して認識を揃える

### 作業中
- 変更を始める前に対象HTMLをバックアップコピーせず、こまめに保存・差分確認を行ってください
- 複数ページへ影響するクラスを変更する場合は影響範囲をメモに残し、着手順を決めてから編集すると安全です

### 作業完了後
- 作業完了後は `style.css` の該当箇所に軽いコメントを残し、次に触る人が意図を把握できるようにしてください

## リニューアル案の背景

### この案にした理由
1. **組織の理念に合致**: 「子供の生涯の幸せシステム」という温かい理念を、デザインで表現できる
2. **父親の参加促進**: 親しみやすいデザインで、これまで関わりが少なかった父親も参加しやすい
3. **実現性**: 開発難易度が比較的低く、現実的な予算・期間で実現可能
4. **メンテナンス性**: シンプルな構造で、継続的な運用がしやすい
5. **レスポンシブ対応**: スマホからのアクセスも快適で、現代のニーズに対応

### 実装済み機能
- トップページとナビゲーション
- 新着情報セクション（既存のコンテンツを移行）
- 各種コンテンツページ
- お問い合わせページ
